# crimeType
Represents the category of cybercrime selected by the user such as financial fraud, identity theft, or cyber bullying. This helps authorities route the complaint correctly.

# description
Allows the user to describe the cybercrime incident in their own words. This provides contextual details required for investigation.

# incidentDate
Captures the date on which the cybercrime incident occurred. Useful for timeline analysis and investigation.

# evidence
Optional field allowing users to upload screenshots, documents, or other proof related to the incident.

# future fields 
- reportId: Unique identifier for each complaint
- status: Tracks the progress of the complaint
